Lisp is a pretty great language. No, really.
It was one of the first high-level languages (like, ever)
It's a fully-functional, extremely regular, & simple. The language is made up of so-called "s-expressions".
S-expressions look something like:
(foo bar (baz foo))
All maths is done in Polish notation (that is to say, 
1 + 1 + 2
is written as 
+ 1 1 2
If you were to write that in an s-expression, it'd look a bit like
(+ 1 1 2)
(+ 1 1 2)
I also added lists and maps (dicts)
[1 2 3 4]
{key1 value1 key2 value2}
So, let's get back to code. Let's take a look at the same code in Python and Lisp.
foo(1) bar(2, 3, "Hello!")
(foo 1) (bar 2 3 "Hello!")
Simple code-branch
if 1 == 1: return x else: return y
(if (== 1 1) x y)
Function declaration
def square(x): return x * x
(defn square [x] (* x x))
Now, let's get down to work here.
Hy is a front-end lexer, and back-end Python AST generator.

This is similar to what Python does (internally) when it takes Python source, and turns it into a .pyc (or imports a module)
AST (or an Abstract Syntax Tree) is how Python represents Python code, inside Python.
(yo dawg)
It's pretty simple, but can be a bit overwhelming. Let's take a look at some AST.
foo()
turns into the AST:
Expr( value=Call( func=Name( id='foo', ctx=Load() ), args=[], keywords=[], starargs=None, kwargs=None ) )
foo("bar")
turns into the AST:
Expr( value=Call( func=Name( id='foo', ctx=Load() ), args=[Str(s='bar')], keywords=[], starargs=None, kwargs=None ) )
if True: return True else: return False
If( test=Name( id='True', ctx=Load() ), body=[ Return( value=Name( id='True', ctx=Load() ))], orelse=[ Return( value=Name( id='False', ctx=Load() ))])
